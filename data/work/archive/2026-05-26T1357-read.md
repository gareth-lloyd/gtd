  Read: "AI is approving our pull requests — here's how we made it safe" (Intercom blog).

  ### TL;DR
  Intercom built a multi-agent AI code reviewer that auto-approves ~19% of PRs, with 6–16x faster time-to-approval and lower revert rates than human-reviewed code (backend AI: 0.53% vs 5.39% human; frontend AI: 0.22% vs 2.00%). Zero reverts in the 100-PR pilot and zero failures across the first 497 auto-approved PRs in broader rollout. Despite doubling deploys, downtime from breaking changes dropped 35%.

  ### Why they built it
  - "Speed is a prerequisite for safety, not its enemy."
  - AI agents now generate working code faster than humans can review it. Without automation, the choice is rubber-stamping or a queue. Both are unsafe.
  - Goal: double R&D productivity in 12 months (hit in 9). PR review surfaced as the bottleneck.
  - Sharp insight: every major past outage at Intercom had been approved by a human. Human review is not the safety guarantee people assume.

  ### Architecture
  Multi-agent decomposition — independent sub-agents per review dimension:
  1. Problem statement quality
  2. Diff-to-intent alignment
  3. Safety concerns
  4. Logical correctness
  5. Best practices / anti-patterns

  Review guidance is Intercom-specific (encoded standards + product context). Engineers flag helpful vs unhelpful comments → guidance updates (feedback flywheel). The agent traces execution paths across the codebase, catching cross-file inconsistencies a human reviewer would only notice if they'd recently touched related code (e.g. caught a copy change that contradicted validation logic elsewhere).

  ### Guardrails
  - **Refuses to auto-approve large/complex/broad-scope PRs** — forces engineers to ship in small increments (incentive design, not just a filter).
  - **Human escape hatch** — any engineer can request human review on any PR, anytime.
  - **Post-merge accountability unchanged** — author still owns prod monitoring + rollback.
  - **Full auditability** — every auto-approved PR labeled, logged, queryable. Comments, decisions, test results, merges all recorded.
  - **Compliance pre-validated** — engaged Schellman (auditor) early; meets SOC 2, HIPAA, ISO 27001, ISO 42001, AIUC-1.

  ### What gets auto-approved
  - ~19% of PRs auto-approved.
  - ~60% reviewed by AI without auto-approval.
  - Only highest-confidence cases get autonomous merge. Exact thresholds not disclosed.

  ### Lessons / pitfalls called out
  - Don't optimize for "% AI-approved" as a metric — gameable, doesn't track to safety.
  - PR review (however rigorous) has a ceiling. Most large historical outages were infra, unanticipated usage, or 3rd-party failures — not code defects. They're building parallel production-diagnostics agents.
  - Humans don't have the mental capacity to review AI-volume output anyway. The honest choice is automate the review or accept rubber-stamping.
